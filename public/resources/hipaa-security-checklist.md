# HIPAA Security Rule Checklist

This checklist is organized by the three safeguard categories defined by the HIPAA Security Rule: Administrative, Physical, and Technical.

## Administrative Safeguards
- Perform and document a risk analysis (45 CFR ยง164.308(a)(1)(ii)(A))
- Implement risk management plan and mitigation actions
- Assign a security official and define roles/responsibilities
- Workforce security: onboarding/offboarding, least privilege
- Information access management: role-based access control (RBAC)
- Security awareness and training program, including phishing
- Contingency plan: backup, disaster recovery, emergency mode ops
- Evaluation: periodic technical and non-technical evaluations
- Business Associate Agreements (BAAs) with all applicable vendors
- Incident response and breach notification procedures

## Physical Safeguards
- Facility access controls and visitor management
- Workstation use and security policies
- Device and media controls: inventory, disposal, re-use procedures
- Environmental and physical protections for server rooms, closets

## Technical Safeguards
- Unique user IDs and strong authentication (SSO/MFA)
- Access control enforcement and session timeouts
- Audit controls: centralized audit logging and retention
- Integrity controls and tamper detection
- Transmission security: TLS 1.2+ in transit; modern cipher suites
- Encryption at rest for PHI (database, object storage, backups)
- Automatic log review and alerting for anomalous activity

## Documentation
- Policies and procedures for each safeguard
- Evidence of controls, tests, and audit logs
- Training records and signoffs
- BAA inventory, data flow diagrams, and system inventories
